d1 CSS framework






Get started



d1 is lightweight yet functional responsive CSS microframework with optional JavaScript enhancements.
It aims to implement most widely used interface components with less code.


Install with NPM


Install to your repository:

npm install d1css


Then use in your CSS file:

@import "../node_modules/d1css/dist/d1.min.css";


And optionally bundle into your JavaScript file:

var d1 = require("d1css");
d1.run();



CDN


To use from CDN just add following code to the head of your page.


<link href="https://cdn.jsdelivr.net/npm/d1css@1/dist/d1.min.css" rel="stylesheet">
<!-- optional -->
<script src="https://cdn.jsdelivr.net/npm/d1css@1/dist/d1.min.js"></script>
<script> d1.run(); </script>

Grid




For the container, add row class.
Direct descendants of the container are columns.
By default, they all have equal width.


To stack columns on small screen, add flip class to the container.


To set various proportions, add following classes to the columns:
col-0 for minimal required width,
col-2 for double width,
col-3 for triple width,
narrow for narrow width,
read to limit maximal width for comfortable reading.


Width limiting classes (col-0, narrow, read) can be combined with:
center to centrify column in its available space,
shift to shift column to the right.


Wrap your rows into div with wrap class to limit maximal width of the whole thing.

Table




Table is styled with border, unless it is a direct descendant of form.
Add hover class to highlight hovered row.
Add r#, l#, #c (where # is any number) classes to align cells of column number # to the right, left or center, respectively (JavaScript required).
Wrap table inside div with class roll to have independent horizontal scrolling of table when it is too wide.

Togglers




Pure CSS togglers based on :target behaviour.
This approach is semantic, but has some drawbacks:

    

  • only one togglable block can be active (open) at the moment

  • nested togglable blocks are not possible

  • styling of active link is not possible

  • on/off behaviour of the link is not possible

  • page scrolls, which is in most cases undesirable




To overcome these drawbacks, JavaScript can be used.
Add toggle class like in following examples for popup, toggle, tree components.
Tabs component is enhanced anyway.
Modals do not require such enhancements.

JavaScript enhancements



    
	
  • Dropdown — toggle, escape
	
  • Tree — multiple, unscroll, toggle, remember, .accordion
	
  • Toggle — multiple, unscroll, toggle, remember
	
  • Popup — multiple, unscroll, toggle, escape
	
  • Tabs — multiple, unscroll, remember
	
  • Dialog — escape
	
  • Gallery — previous, escape
	
  • Output HTML code (pre[data-src])
	
  • Link or button with confirmation (.confirm[title])
	
  • Link with prompt (a.prompt[title][data-default])
	
  • Check group of boxes (input[data-group], [class~="data-group"])
	
  • Align table columns (table.r5)
	
  • Show dropped uploaded image (input.drop)

Alphabetical





    

  1. .act — active link

  2. .bg — light background

  3. .bg-e — nominal "red" background

  4. .bg-i — nominal "blue" background 

  5. .bg-n — nominal "gray" background

  6. .bg-w — nominal "yellow" background

  7. .bg-y — nominal "green" background

  8. .bord — add border

  9. .c — centrify content inside block

  10. .center — centrify block itself

  11. .close — link to close togglable block

  12. .col-0 — grid cell with non-expanding width

  13. .col-2 — grid cell with expanding width (x2)

  14. .col-3 — grid cell with expanding width (x3)

  15. .dlg — card-like block

  16. .drop — file input to show dragged image

  17. .flip — responsive component, stacked view on small screen

  18. .full — fullscreen block

  19. .gal — fullscreen gallery

  20. .hide — hidden

  21. .hover — highlight link or table row on hover

  22. .icon — small image

  23. .js-control — visibility is controlled by JavaScript, hidden by default

  24. .js-show — shown with JavaScript

  25. .l — align content to the left

  26. .mar — add vertical margins

  27. .narrow — narrow block width

  28. .nav — horizontal dropdown navigation

  29. .pad — add padding

  30. .pop — popup component wrapper

  31. .print — show only on print (not on screen)

  32. .r — align content to the right

  33. .rad — rounded corners

  34. .read — block width for comfortable reading

  35. .roll — horizontal scrolling of contents

  36. .row — row of grid

  37. .screen — show only on screen (not on print)

  38. .shift — shift item of .row or .nav to the right

  39. .show-desktop — force visibility on large screen (overrides .hide)

  40. .show-mobile — force visibility on small screen (overrides .hide)

  41. .small — small font size

  42. .stick — stick block to the top

  43. .switch — input controlling visibility of following block

  44. .tabs — tabs component

  45. .text-e — nominal "red" text color

  46. .text-i — nominal "blue" text color

  47. .text-n — nominal "gray" text color

  48. .text-w — nominal "yellow" text color

  49. .text-y — nominal "green" text color

  50. .topbar — sticky top app bar

  51. .wrap — reasonable page width limit

  52. .yel — temporary highlight targeted block
